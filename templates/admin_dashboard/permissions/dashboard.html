Berechtigungskonzept
Unser dreistufiges Zugriffsmanagement:
  1. Rollen (Gruppen) - Bestimmen grundlegende Funktionen Beispiel: "Einkäufer" können Bestellungen anlegen
  2. Lager-Zugriffsrechte - Bestimmen Arbeitsbereiche für Abteilungen Beispiel: "Marketing-Abteilung" kann auf "Marketing-Materiallager" zugreifen
  3. Objektberechtigungen - Für Ausnahmen und Sonderfälle Beispiel: "Vertrauliches Produkt XYZ" darf nur von bestimmten Personen gesehen werden
Entscheidungshilfe für Berechtigungen:
Frage 1: Welche Arbeitsrolle hat der Benutzer?
→ Rolle/Gruppe zuweisen
Frage 2: Auf welche Lager soll die Abteilung des Benutzers zugreifen können?
→ Warehouse Access konfigurieren
Frage 3: Gibt es spezielle Einzelobjekte, die von den Standardregeln abweichen?
→ Object Permissions einsetzen
